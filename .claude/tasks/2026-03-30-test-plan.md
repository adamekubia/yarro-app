## How to Use This Plan

Work through phases in order — each phase builds on the last.

**Bug triage rule:** When you find an issue, categorize it immediately:

| Bucket | Definition | Action |
|--------|-----------|--------|
| **BLOCKER** | Breaks the current scenario or prevents testing the next one | Stop. Fix it now. |
| **VISUAL** | Wrong spacing, color, label, alignment — doesn't break function | Log it. Batch fix in Phase 5. |
| **EDGE** | Only triggers in unusual conditions unlikely during demo | Log it. Fix post-demo. |

**The rule: if a bug blocks the next test, fix it now. If it doesn't, log it and keep moving.**

### 0d. Seed Data Audit

Run these queries in Supabase SQL Editor (or via `psql`) to verify demo data exists:

```sql
-- Property exists
SELECT id, address FROM c1_properties
WHERE address ILIKE '%brixton%';

-- 5 rooms exist
SELECT r.room_number, r.room_name, r.monthly_rent,
       t.full_name as tenant
FROM c1_rooms r
LEFT JOIN c1_tenants t ON t.id = r.current_tenant_id
WHERE r.property_id = (SELECT id FROM c1_properties WHERE address ILIKE '%brixton%')
ORDER BY r.room_number;

-- Compliance certs exist with mixed statuses
SELECT certificate_type, expiry_date,
  CASE
    WHEN expiry_date IS NULL THEN 'missing'
    WHEN expiry_date < CURRENT_DATE THEN 'expired'
    WHEN expiry_date < CURRENT_DATE + interval '30 days' THEN 'expiring'
    ELSE 'valid'
  END AS status
FROM c1_compliance_certificates
WHERE property_id = (SELECT id FROM c1_properties WHERE address ILIKE '%brixton%')
ORDER BY expiry_date;

-- Rent ledger entries exist (or note that we need to generate them)
SELECT rl.due_date, rl.amount_due, rl.status, r.room_number, t.full_name
FROM c1_rent_ledger rl
JOIN c1_rooms r ON r.id = rl.room_id
LEFT JOIN c1_tenants t ON t.id = rl.tenant_id
ORDER BY rl.due_date, r.room_number;
```
